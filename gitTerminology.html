Git Terminology

Version Control Software
Version Control, also known as source or revision control, is the monitoring, organization and management of changes to a base of information, particularly computer programs, web sites and applications, and documents. It is essential to multi-editor projects, and greatly beneficial to individual endeavors as well.
Git
Git is a distributed revision control system that supports non-linear development and is used for software and application development. It was developed in 2005 by Linus Torvalds and is maintained by Junio Hamano, and now is the most widely used source code management system among software developers.
Repository
A repository is where files' current and historical data is stored. This can be placed on a server or placed in a shared cloud service, such as BitBucket
Stage
A stage is an index or a cache under constant change and is used to store information about the working directory as well as the changes being made to it that will become the next committed revision. It is a connection point between the working tree and object database.
Commit
To commit is to write or merge the changes from the working copy of a file into the repository
Push
A push is to copy revisions initiated by the source repository into a recieving or user's repository
Pull
The opposite of a push, a pull is a user initiated copy of revisions into a repository.
Revert
To revert is to move backwards in historical data and changes to a version of the repository prior to the current commit.
Branching
Branching is pulling a repository and editing the code independently. Once the code is sufficiently altered, it is merged back into the repository
Merging
Merging or integration is the point where sets of changes are evaluated and applied accordingly to a set of files in the repository.
Change History
The change history is the list of modifications performed on the data or code in the file or repository.
Clone/Cloned
A clone is a duplication of data and revisions from one repository into another. If pushing or pulling a populated or active repository into an empty or newly created one, that is considered a clone. Two repositories that have identical revisions are also considered to be clones.
